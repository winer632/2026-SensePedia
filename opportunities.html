大模型应用阶段 01

大模型应用的三个阶段

Phase 1
会回答
2022 - 2023
ChatGPT, 文心一言, 通义千问
大模型从实验室走向大众，核心能力是"对话式问答"。面向C端用户，交互形态是聊天窗口。
Phase 2
会执行
2024 - 2025
Claude Code, Manus, OpenClaw, 豆包手机
从"回答问题"进化到"执行任务"。Agent 能调用工具、操作系统、编排工作流。仍以C端开发者和个人用户为主。
Phase 3
会经营
2026 →
SensePedia Agent Platform
将 Agent 能力注入政企核心业务流。不是替代现有系统，而是成为跨系统工作流的交互层和编排层——政企客户的超级入口。
Key Insight

第一、第二阶段的产品都面向 C 端用户。toB 市场存在巨大空白——这正是 SensePedia Agent Platform 的机会窗口。

制造业场景预览 03

跨系统工作流编排：Agent 最有价值的切入点

Scenario Preview

采购经理在飞书问 Agent："昨天那批电容到货了吗？"

Agent → SRM 查物流状态 → ERP 查入库记录 → QMS 查来料检验

30 秒返回跨系统状态报告

基于平台架构设计的场景预览，尚未对接真实系统

Agent 高价值切入点

采购跟催 SRM↔ERP 质检报告 QMS→ERP 订单状态 CRM→ERP→MES→WMS

制造业全链路工作流

CRM → CAD → PLM → ERP → SRM → MES → QMS → WMS/TMS

高亮节点为 Agent 优先切入点：流程跨系统、查询频次高、痛感强、可闭环

制造业工作流最长（8 个核心系统）、跨角色协同最强——Agent 编排价值最高

现实挑战 06

真正难的不是模型，而是把 Agent 嵌入真实业务流程

01
系统异构集成成本高
先做 API 已开放的系统，积累连接器资产
02
安全合规责任边界高
关键操作保留人工确认 + 全链路审计
03
数据质量不统一
先做"查询展示"不做"自动执行"
04
商业生态不配合
先找实施伙伴和国产厂商合作
05
产品化 vs 项目化张力
短期项目制验证，中期提取平台
06
客户组织变革阻力
把 IT 变成共建者，不说"替代"说"增强"
以上应对方向是初步判断，每一项都需要通过实际客户验证
讨论 07

客户为什么会为这类 Agent Platform 买单？

Discussion · 请从各自角色出发给观点
背景
  • 制造业 IT 预算被 SAP/MES/CRM 锁定
  • Agent Platform 的预算从哪里出？
  • 客户已有 RPA 在跑，为什么要换？
种子观点
  • 不卖"入口"，卖"效率提升"——采购跟催从 2 小时变 5 分钟
  • 可能的预算来源：数字化转型专项 / 降本增效专项 / IT 创新预算
  • 商业模式候选：项目制（短期）→ 订阅制（中期）→ 平台+分成（长期）
以上为讨论假设，尚无客户验证
讨论 08

售前第一次见客户，应该怎么讲，才不会被理解成"更贵的集成项目"？

Discussion · 请从各自角色出发给观点
差异化定位
vs 钉钉/飞书 AI

他们做办公协同+AI增强，我们做行业工作流级 Agent 编排——对接 ERP/MES/PLM，不是聊天和审批

vs Copilot

绑微软生态（Office+Azure），我们做跨厂商跨系统的行业编排层——不绑定特定生态

vs Coze / Dify

通用 Agent 开发平台，我们做行业纵深——沉淀制造业 skill 库而非通用工具

以上差异化方向待验证，Copilot connector 生态已很强
PoC 思路
  1. 最容易打动客户的 PoC：跨系统订单状态查询（客户每天在做，痛感强）
  2. PoC 周期挑战：需要客户开放测试环境 API
  3. 单系统快速对接：先对接客户已有 Open API 的单个系统（如 ERP 查询），验证 Agent 交互体验后再扩展到跨系统
  4. 文件级数据分析切入：客户上传 Excel/CSV 业务数据，通过小浣熊 Skill 直接分析出图表和洞察——零系统对接、当场出效果
讨论 09

我们现在应该优先做行业方案、平台能力，还是两者结合？边界怎么定？

Discussion · 请从各自角色出发给观点
核心张力
  • P 主攻平台 PaaS：客户是开发者/ISV，离行业场景远
  • S 主攻行业方案：每个客户定制，很难规模化
当前假设：先做 1 个行业（制造业）闭环，在交付中沉淀平台能力
需要碰撞的子问题
  1. 哪些 skill 能跨客户复用？哪些必须定制？
  2. 第一个客户的交付边界画在哪？
  3. 团队需要什么新能力（行业顾问？系统对接工程师？）
  4. 在巨头（Copilot / SAP Joule / Salesforce Agentforce）之间，我们的生存空间在哪？
初步判断：行业纵深 + 私有化部署 + 本地化服务
收口 10

今天不追求完整共识，而是把关键问题摆上桌面

一句话机会
ToB 机会在于成为跨系统工作流的交互层和编排层
一句话现实
最大的难点不是模型，而是集成、合规、数据和可复制性
一句话下一步
选一个具体的制造业场景，定义"最小可卖方案"，启动内部 workshop

我的判断不是企业软件会被立刻替代，而是企业软件的使用方式会被重构。但路径上必须克制，先从高价值、可闭环的流程切入。
